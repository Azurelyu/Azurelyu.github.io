给个人博客换上专属的免费域名

【github+hexo+freenom+dnspod】

今天真是波折的一天，折腾了一天，终于把各种都准备好了，域名什么的也都完成了。很感谢百度2333，让我有机会找到那么多教程。好了，废话不多说，开干！

这一切的前提是搭建好了一个部署在githubpage上的个人博客，比如参考我之前写的文章。

第一步：申请一个免费域名

免费域名申请网站freenom,直接输入自己想要的名字，然后可以在搜索结果界面找到免费的，注意，它默认是3个月，记得改成12个月。并且，需要科学上网，而且需要将网站的个人资料上的地址等改成美国的，可以用这个网站好维持生成一个虚假身份。

我使用的是DNSPOD，腾讯云的一款产品。

三、Hexo

​ 不准备用Wordpress了，用hexo。

1.hexo的配置环节

​ 基于mac下完成。需要的有homebrew、nodejs、git。

​ 其中，git一般mac都是自带的，homebrew和nodejs都可以去官方网站下载。homebrew以及nodejs完成之后，就需要来配置hexo，可以去hexo.io官网，看到npm install hexo-cli -g。这个时候就完成了对hexo框架的下载。

​ 下一步就是在本地新建一个文件夹，例如/usr/blog。terminal进入到该文件夹中，执行命令hexo init，待命令执行完成之后，从finder进入文件夹可以看到文件夹中已经完成了整个框架的建立。回到terminal，执行hexo g即generate再执行hexo s即start，可以看到本地的localhost:4000中已经有了基础的界面，还为你写好了第一篇博文“Hello World”。

​ 接下来我们肯定是需要将博客放到网上，这里我使用免费的github来完成，登陆自己的github账号，新建一个库，注意，库名一定要是自己github的ID.github.io。例如我的账号为Azurelyu，所以我的库名就是Azurelyu.githu.io。回到本地，我们回到blog文件夹，这个时候需要用npm install --save hexo-deployer-git来完成我们与github的交互。修改blog文件夹下的_config.yml的最下面的

注意，下面的三个冒号后面有一个空格。回到blog文件夹hexo clean hexo g hexo d执行完这三步之后可以看到博客已经上传到Azurelyu.github.io上了。

​ 以上就是hexo的最基础部署。
